Oriel is a lightweight jQuery plugin for image slideshows. Rather than use JavaScript functions for transition effects, Oriel leaves that to the CSS (the JavaScript essentially organizes the DOM structure and adds CSS classes).

A very basic stylesheet for Oriel that uses opacity and transition to achieve fade in/out might look like: 

.oriel-stage {
  height: 500px; // Need to define a height
}
.oriel-stage img.oriel-active {
  opacity: 1.0;
}
.oriel-stage img {
  opacity: 0.0; // Need to hide images by default
  position: absolute;
  transition: opacity 0.2s ease-in-out;
}

Oriel is easy to hook into. It uses standard jQuery events: 

var $oriel = $('.images').oriel();

$oriel.bind('beforeNext', function(e) {
  var oriel = e.oriel;
  oriel.setCaption("Image no. " + oriel.currentIndex);
});

It also doesn’t force you to use any particular DOM structure. Let’s say you had the (strange) HTML: 

<div id="slideshow">
  <p data-full-src="images/full-image-1.jpg">
    <a href="images/thumb-image-1.jpg">Image 1</a>
  </p>
  <p data-full-src="images/full-image-2.jpg">
    <a href="images/thumb-image-2.jpg">Image 2</a>
  </p>
</div>

On calling $.fn.oriel you may pass in an option hash with functions such as getCaption and getFull that parse the DOM appropriately: 

$('#slideshow').oriel({
  itemSelector : 'p',
  getCaption : function (el) {
    return el.find('a').text();
  },
  getFull : function (el) {
    return el.data('full-src');
  },
  getThumb : function (el) {
    return el.find('a').attr('href');
  }
});

This is useful in systems that predefine the HTML structure (such as Drupal).

Check out the demos and documentation, and I also recommend reading the annotated source.
